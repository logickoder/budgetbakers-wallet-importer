## Auth Endpoints

Base URL: `https://web.budgetbakers.com/api`

All tRPC calls use batch format: the response is an array; result lives at `[0].result.data.json`.

### 1. Trigger SSO Email

```
POST /trpc/user.ssoSignInEmail?batch=1
Content-Type: application/json

{ "0": { "json": "user@example.com" } }
```

**Response:** `[{ "result": { "data": { "json": "<ssoKey>" } } }]`

The server sends a login link to the email address. The `ssoKey` is opaque — pair it with the token from the email.

---

### 2. Exchange Tokens for Auth Token

```
POST /trpc/user.confirmSsoAuth?batch=1
Content-Type: application/json

{ "0": { "json": { "ssoKey": "...", "ssoToken": "...", "userEmail": "..." } } }
```

**Response:** `[{ "result": { "data": { "json": "<authToken>" } } }]`

`ssoToken` is the UUID from the login link: `https://web.budgetbakers.com/sso?ssoToken=<here>`

---

### 3. Get CSRF Token

```
GET /auth/csrf
```

**Response:** `{ "csrfToken": "<token>" }`

Required by Next-Auth for the session callback POST.

---

### 4. Complete Session (Next-Auth Callback)

```
POST /auth/callback/sso
Content-Type: application/x-www-form-urlencoded

token=<authToken>&csrfToken=<csrfToken>&callbackUrl=https://web.budgetbakers.com
```

**Response:** HTTP 302 with `Set-Cookie: __Secure-next-auth.session-token=<jwt>`

Set `maxRedirects: 0` to capture the cookie before the redirect.

---

### 5. Get User Profile + CouchDB Credentials

```
GET /trpc/user.getUser?batch=1&input={"0":{"json":null,"meta":{"values":["undefined"]}}}
Cookie: __Secure-next-auth.session-token=<jwt>
```

**Response (relevant fields):**
```json
{
  "userId": "8d30a9fd-a575-4f41-82f4-3212bcd4789e",
  "email": "user@example.com",
  "replication": {
    "dbName":  "bb-8d30a9fd-a575-4f41-82f4-3212bcd4789e",
    "url":     "https://couch-prod-eu-2.budgetbakers.com",
    "login":   "8d30a9fd-a575-4f41-82f4-3212bcd4789e",
    "token":   "<stable-uuid>",
    "ownerId": "8d30a9fd-a575-4f41-82f4-3212bcd4789e"
  }
}
```

The `replication` block is the **single source of CouchDB credentials** — no separate call needed.

## CouchDB Access

**Base URL:** `https://couch-prod-eu-2.budgetbakers.com/bb-{userId}/`

**Auth:** `Authorization: Basic base64("{replication.login}:{replication.token}")`

The `login` and `token` are taken from the `replication` block in `user.getUser`. The token is a stable UUID that does not rotate per session.

**Quick test:**
```bash
curl "https://couch-prod-eu-2.budgetbakers.com/bb-{userId}/" \
  -H "Authorization: Basic {base64(login:token)}"
```

## CouchDB Document Types

Confirmed from `_all_docs` key prefixes:

| Prefix        | Type            | Notes                                       |
| ------------- | --------------- | ------------------------------------------- |
| `-Account_`   | Account         | Leading `-` sorts accounts before records   |
| `-Budget_`    | Budget          | Not used by the importer                    |
| `-Category_`  | Category        |                                             |
| `-Configure_` | App config      | Not used                                    |
| `-Currency_`  | Currency        |                                             |
| `-Debt_`      | Debt tracking   | Not used                                    |
| `-Filter_`    | Saved filters   | Not used                                    |
| `-Goal_`      | Savings goals   | Not used                                    |
| `-HashTag_`   | Labels/hashtags | Referenced in `labels` on records           |
| `Record_`     | Transaction     | No leading `-`, sorts after everything else |

## Record Field Reference

All fields confirmed from 20+ real Record documents. No fields are assumed.

```typescript
{
  // Identity
  _id:                 "Record_<uuid>",   // generated by writeRecords()
  _rev:                "1-<hash>",         // CouchDB revision — omit when creating

  // Classification
  type:                0 | 1,              // 0=income, 1=expense (BOTH confirmed)
  recordState:         1,                  // always 1
  categoryChanged:     true,               // always true on real records

  // Account & money
  accountId:           "-Account_<uuid>",  // full id — NOT uuid-only
  currencyId:          "-Currency_<uuid>", // full id — NOT "NGN" string
  amount:              5375,               // POSITIVE integer, minor units ×100
  refAmount:           5375,               // always same as amount

  // Category
  categoryId:          "-Category_<uuid>", // full id

  // Description
  note:                "Stamp Duty",
  payee:               "Chowdeck",         // SEPARATE field — not embedded in note

  // Time
  recordDate:          "2026-01-27T02:31:00.000+01:00",  // full ISO-8601 with local offset

  // Payment
  paymentType:         3,                  // 0=cash, 3=electronic transfer

  // Transfer linking (inter-account transfers only)
  transfer:            true,               // boolean, ALWAYS present
  transferId:          "<shared-uuid>",    // same UUID on both legs of the pair
  transferAccountId:   "-Account_<uuid>",  // the OTHER account's full id

  // Always zero/empty
  latitude:            0.0,
  longitude:           0.0,
  accuracy:            0,
  warrantyInMonth:     0,
  suggestedEnvelopeId: 0,
  photos:              [],
  refObjects:          [],
  labels:              [],                 // or ["-HashTag_<uuid>"]

  // Metadata
  reservedModelType:   "Record",
  reservedSource:      "backend" | "web",
  reservedOwnerId:     "<userId>",
  reservedAuthorId:    "<userId>",
  reservedCreatedAt:   "<ISO timestamp>",
  reservedUpdatedAt:   "<ISO timestamp>",
}
```

### `type` values

| Value | Meaning | Confirmed                                |
| ----- | ------- | ---------------------------------------- |
| `0`   | Income  | ✓ Record_009fd34a "Capitalised Interest" |
| `1`   | Expense | ✓ multiple records                       |

### `paymentType` values

| Value | Meaning                  | Confirmed                   |
| ----- | ------------------------ | --------------------------- |
| `0`   | Cash                     | ✓ Records: Transportation   |
| `3`   | Electronic/bank transfer | ✓ Records: Stamp Duty, HDMI |

### Transfer pair structure

Both legs of an inter-account transfer (e.g. First Bank → Palmpay) have:
- `transfer: true`
- `transferId`: the same shared UUID
- `transferAccountId`: the OTHER account's full `-Account_<uuid>`
- `categoryId`: `-Category_16b2df8a-...` ("Transfer, withdraw")

## Lookup Queries

All queries use `_all_docs` with key range. Auth header required on all.

### Fetch Accounts

```bash
curl "{baseUrl}/_all_docs?include_docs=true&startkey=%22-Account_%22&endkey=%22-Account_%EF%BF%BD%22" \
  -H "Authorization: Basic {credentials}"
```

Account `name` field matches the CSV `account` column exactly.
The full `_id` (e.g. `-Account_15230f1e-...`) is what goes in `accountId` on records.

### Fetch Categories

```bash
curl "{baseUrl}/_all_docs?include_docs=true&startkey=%22-Category_%22&endkey=%22-Category_%EF%BF%BD%22" \
  -H "Authorization: Basic {credentials}"
```

Category `name` mostly matches the CSV `category` column. Exception:
- CSV `"TRANSFER"` → CouchDB name `"Transfer, withdraw"`

### Fetch Currencies

```bash
curl "{baseUrl}/_all_docs?include_docs=true&startkey=%22-Currency_%22&endkey=%22-Currency_%EF%BF%BD%22" \
  -H "Authorization: Basic {credentials}"
```

Currency `code` field is the ISO 4217 string (e.g. `"NGN"`, `"USD"`).
The full `_id` (e.g. `-Currency_38481cb0-...`) is what goes in `currencyId` on records.

### Fetch Records (for inspection)

```bash
# First 20 records
curl "{baseUrl}/_all_docs?include_docs=true&startkey=%22Record_%22&endkey=%22Record_%EF%BF%BD%22&limit=20" \
  -H "Authorization: Basic {credentials}"

# Specific record
curl "{baseUrl}/Record_0e092dd8-2bb2-4995-9d61-7ca0a1172c5a" \
  -H "Authorization: Basic {credentials}"
```

## Writing Records

### Single `_bulk_docs` call

```
POST {baseUrl}/_bulk_docs
Authorization: Basic {credentials}
Content-Type: application/json

{
  "docs": [
    {
      "_id":                "Record_<new-uuid>",
      "type":               1,
      "accountId":          "-Account_15230f1e-31fd-400a-a4c3-210c11d9358a",
      "currencyId":         "-Currency_38481cb0-8314-41fe-b8ce-fd0ff87758be",
      "categoryId":         "-Category_8013274a-7535-4d96-ac71-3a938ea22903",
      "amount":             5375,
      "refAmount":          5375,
      "note":               "Stamp Duty",
      "recordDate":         "2026-01-27T02:31:00.000+01:00",
      "recordState":        1,
      "paymentType":        3,
      "transfer":           false,
      "categoryChanged":    true,
      "latitude":           0.0,
      "longitude":          0.0,
      "accuracy":           0,
      "warrantyInMonth":    0,
      "suggestedEnvelopeId": 0,
      "photos":             [],
      "refObjects":         [],
      "labels":             [],
      "reservedModelType":  "Record",
      "reservedSource":     "web",
      "reservedOwnerId":    "<userId>",
      "reservedAuthorId":   "<userId>",
      "reservedCreatedAt":  "2026-03-22T10:00:00.000Z"
    }
  ]
}
```

**Response:** `[{ "id": "Record_...", "rev": "1-...", "ok": true }]`

CouchDB returns HTTP 201 even on partial failures — always check each entry for `error`.

## Custom CSV Format

This is the format the importer expects. It is simpler than the official Wallet export because the user is filling it in manually.

```csv
date,account,amount,category,note,payee
2026-01-27 02:31:00,First Bank,-53.75,Charges & Fees,Stamp Duty,
2026-01-29 13:33:00,First Bank,-300000,Transfer,,
2026-01-29 13:33:00,Palmpay,300000,Transfer,,
2026-02-10 11:25:00,First Bank,300000,Wage & invoices,,Company XYZ
2026-02-28 15:00:00,First Bank,-25020,Restaurant & fast-food,Chowdeck,
```

### Column rules

| Column     | Notes                                               |
| ---------- | --------------------------------------------------- |
| `date`     | `YYYY-MM-DD HH:MM:SS` — interpreted as local time   |
| `account`  | Exact name as shown in the app                      |
| `amount`   | Signed float. Negative = expense, positive = income |
| `category` | Exact name as shown in the app                      |
| `note`     | Optional free text                                  |
| `payee`    | Optional — stored as a separate field on the record |

### What you don't fill in

| Field          | How it's derived                                                           |
| -------------- | -------------------------------------------------------------------------- |
| Currency       | Taken from the account's own currency in CouchDB                           |
| Transfer flag  | Set when category is "Transfer, withdraw" + both rows share same timestamp |
| Payment type   | Transfer rows → 3 (electronic), all others → 0 (cash)                      |
| Income/expense | Sign of `amount`                                                           |

### Transfer pairs

Write two rows with the same `date` and category "Transfer, withdraw" (or whatever you named it in the app). The importer links them automatically with a shared `transferId`.

```csv
2026-01-29 13:33:00,First Bank,-300000,Transfer,,
2026-01-29 13:33:00,Palmpay,300000,Transfer,,
```

Both conditions must match for linking: same category **and** same timestamp. This prevents false pairing of coincidentally same-minute rows with different categories.
